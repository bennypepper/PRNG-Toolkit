About The PRNG Toolkit

This tool was built by Benedict Pepper. It originated as a vital testing hub during a university Modeling and Simulation course. The end project was a robust Queueing System Simulator (which you can check out on my GitHub).

Before making the simulation, it is critical to guarantee that the generated numbers representing inter-arrival times or queue probabilities are statistically robust. This application is designed to educate on how different algorithms work and rigorously verify their entropy via Hypothesis Testing.

Algorithm Glossary

Mersenne Twister (MT19937)

The default PRNG for Python, C++, and Excel. It has a massive period length of $2^{19937}-1$. Uses deeply complex bitwise twisting matrices.

Period: 2^19937-1

Permuted Congruential (PCG32)

A highly modern, statistically brilliant algorithm. It utilizes LCG state transition but scrambles the output via an XSH-RR permutation layer.

Recommended for simulations

Linear Congruential (LCG)

One of the oldest PRNG algorithms. Very fast, but flawed if parameters ($a, c, m$) aren't well chosen via the Hull-Dobell theorem.

Formula: X_{n+1} = (aX_n + c) % m

Middle-Square

Devised by John von Neumann. You square a seed, extract the middle digits, and repeat. Poor periods, prone to collapsing to zero quickly.

Status: Educational only

Statistical Testing Glossary

Null Hypothesis (H₀)

Hypothesis testing determines if a sequence differs significantly from "true randomness". H₀ always assumes the generated numbers *are* truly random. If the P-Value drops below our threshold (commonly $\alpha = 0.05$ or $0.01$), we Reject H₀ (Meaning the sequence is predictable/flawed).

  • Frequency (Monobit): Counts binary ones vs zeros. They should be roughly 50/50.
  • Runs Test: Analyzes oscillation. If a sequence goes 0,0,0,0,0,0 it's too clustered. If it goes 1,0,1,0,1,0 it's too predictable. Runs ensures grouping happens naturally.
  • Chi-Square Uniformity: Drops numbers into 10 buckets. It calculates if buckets are filled equally, proving the distribution is uniform.
  • Autocorrelation: Tests if generating a number mathematically biases the next number generated.
  • Poker Test: Simulates drawing poker hands from digits (e.g. "Three of a kind") matching predicted categorical probabilities.
