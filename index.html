Information & Best Practices

About This Program

This application is a tool for exploring the world of pseudo-random number generation (PRNG). It allows you to create sequences of numbers using different mathematical algorithms and then analyze their quality using formal statistical tests. The goal is to understand that "random" numbers generated by computers are not truly random, but can be "good enough" for many purposes if the generator is well-designed.

Tips for Generator Parameters

The quality of your generated numbers depends heavily on the initial parameters you provide.

  • Seed: The seed is the starting point for the generator. The same seed will always produce the exact same sequence of numbers. For good results, a seed should be unpredictable.
    • Middle-Square Tip: This method is very sensitive to the seed. Generally, the longer the seed (more digits), the longer the sequence will be before it repeats or degenerates (e.g., starts producing all zeros). A 4-digit seed is a minimum for experimentation; 6 or 8 digits is better.
    • Real-World Sources for Seeds: To get a more unpredictable seed, you can use real-world events. For example, start a stopwatch and stop it randomly; use the milliseconds (e.g., 1438 ms) as your seed. Other sources include mouse movement coordinates or timing the pauses between your keystrokes. The "Auto" button uses the current system time in milliseconds, which is a common and decent source of unpredictability.
  • LCG Parameters (a, c, m): To get the longest possible sequence before numbers repeat (a "full period"), these parameters must satisfy the Hull-Dobell Theorem. The presets provided (like "Numerical Recipes") are known to have good properties.
  • Xorshift: This algorithm is generally very fast and produces high-quality random numbers. It is less sensitive to its parameters than LCGs, so the main focus is on providing a good, unpredictable seed.

Understanding Randomness

What makes a sequence of numbers "random"? Two key properties are tested:

  • Uniformity: The numbers should be evenly spread across their possible range. For example, in a sequence from 0-99, the number of values between 0-9 should be roughly the same as the number between 90-99. The Chi-Square test and Histogram visualization check for this.
  • Independence: Knowing one number in the sequence should give you no information about the next number. There should be no discernible pattern or correlation. The Autocorrelation and Runs tests check for this.

Pseudo-Random vs. True Random

The algorithms in this program are Pseudo-Random Number Generators (PRNGs). They are deterministic algorithms that produce sequences that only *appear* random. In contrast, True Random Number Generators (TRNGs) use unpredictable physical phenomena as a source of data, such as atmospheric noise, radioactive decay, or cosmic background radiation. While PRNGs are sufficient for simulations and games, TRNGs are required for high-stakes applications like cryptography.
